# BirdLink App

这是一个基于《信息采集与变送器的硬件功能要求》整理出的前端 App 原型，所有文件都在当前 `App` 文件夹内。

## 打开方式

直接用浏览器打开 `index.html` 即可运行。

如果要让手机像 App 一样安装到桌面，需要把整个文件夹部署到 HTTPS 网站，然后在手机浏览器中选择“添加到主屏幕”。当前已包含：

- `manifest.json`：App 名称、图标、主题色、全屏模式。
- `service-worker.js`：离线缓存。
- `icon.svg`：应用图标。

## 已覆盖的功能

- 设备总览：在线状态、WiFi、电量、模式、探针状态、状态灯。
- WiFi 配网：设备热点、WiFi 名称、密码写入、连接结果展示、支持更换 WiFi。
- 传感器采集：土壤电离度、氮磷钾、pH、湿度、温度、光照、人体感应。
- 采集逻辑：每 2 小时自动采集开关、立即采集、人体感应上报。
- 趋势图：湿度、温度、pH、光照历史趋势。
- 语音交互：上传语音、服务器返回语音与指令、音量和静音管理。
- 设备控制：休眠、勿扰定时、状态灯、探针连接、重启、恢复默认配置。
- 告警提醒：低电量、WiFi 断开、探针断开、数据异常、人体感应事件。
- 设备信息：产品 ID、ESP32、Type-C、电池、防护等级、上传下载内容、协议、固件和认证事项。

## 后续接硬件/服务器时需要替换

当前 `app.js` 中的数据和指令是本地模拟。后续可以把这些函数替换为真实接口：

- `collectSensors()`：调用服务器或设备的立即采集接口。
- `wifiForm` 提交逻辑：调用 ESP32 配网接口。
- `addCommand()`：改成真实指令下发。
- `state.sensors` 和 `state.history`：改成服务器返回的数据。

## 关于 APK

如果要生成 Android 安装包 `.apk`，需要安装 Android Studio / Android SDK。安装好后可以用 Capacitor 把这个 Web App 包成原生 Android App。
